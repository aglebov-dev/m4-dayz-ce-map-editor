Программа использует библиотеки ниже. Они распространяются на собственных условиях;
ограничения раздела 2 [LICENSE](LICENSE) на них не распространяются.

| Библиотека | Версия | Лицензия | Входит в сборку exe |
|---|---|---|---|
| PySide6 (+ PySide6-Essentials, shiboken6) | 6.11.1 | LGPL-3.0-only OR GPL-2.0-only OR GPL-3.0-only | да |
| numpy | 2.5.1 | BSD-3-Clause (+ 0BSD, MIT, Zlib в составе) | да |
| Pillow | 12.3.0 | MIT-CMU | да |
| paramiko | 5.0.0 | LGPL-2.1 | да (загрузка по SFTP) |
| cryptography | 49.0.0 | Apache-2.0 OR BSD-3-Clause | да (зависимость paramiko) |
| bcrypt | 5.0.0 | Apache-2.0 | да (зависимость paramiko) |
| PyNaCl | 1.6.2 | Apache-2.0 | да (зависимость paramiko) |
| Nuitka | 4.x | Apache-2.0 | нет — только инструмент сборки |

Тексты лицензий поставляются вместе с самими пакетами (в каталогах `*.dist-info`
виртуального окружения) и доступны на страницах проектов.

## Обязательства по LGPL

**PySide6** и **paramiko** распространяются на условиях LGPL. Если сборка передаётся
третьим лицам, LGPL требует дать получателю возможность **заменить** эти библиотеки
собственной версией и не препятствовать действиям, необходимым для такой замены.

LGPL-3.0 (§4d) даёт на выбор два способа это обеспечить: либо динамическая линковка с
заменяемой библиотекой, либо предоставление исходников и возможности **пересобрать**
приложение с изменённой версией библиотеки. Мы идём вторым путём.

**Как заменить LGPL-компонент.** Разрешение на это дано прямо в разделе 3.1
[LICENSE](LICENSE), пункты 2.2–2.4 к таким действиям не применяются.

```powershell
git clone <официальный репозиторий>          # исходники программы
.\rebuild-env.bat                            # окружение по requirements.txt
.venv\Scripts\pip install --force-reinstall <своя сборка PySide6 / paramiko>
.\deploy.ps1                                 # сборка с вашей версией библиотеки
```

Версии зависимостей закреплены в `requirements.txt`, параметры сборки — в
`src/pysidedeploy.spec`; ничего скрытого в процессе нет. Полученную сборку можно
использовать на условиях раздела 1 лицензии, распространять — нельзя.

**Что кладём рядом со сборкой.** LGPL требует уведомить получателя об используемых
библиотеках и приложить тексты лицензий. Поэтому в архив релиза (см.
`.github/workflows/build-windows.yml`) вместе с exe кладутся `LICENSE` и этот файл.

**Режим сборки.** При `mode = standalone` библиотеки лежат рядом с exe отдельными
файлами и заменяются без пересборки. При `mode = onefile` (текущая настройка) всё
упаковано в один файл, поэтому путь замены — именно пересборка, описанная выше.

## Данные игры

Спутниковые тайлы, датасеты зданий и файлы миссий, которые программа извлекает из
файлов DayZ, остаются собственностью Bohemia Interactive a.s. и не распространяются.
В репозитории их нет — они создаются локально в `appdata/` на машине пользователя.
