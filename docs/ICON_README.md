# Иконки приложения (семейство M4)

Иконки генерируются векторно-плоско скриптом `create_icon.py` (Pillow, без внешних
зависимостей). Стиль — «Объём»: градиентная squircle-плитка + верхний блик +
монограмма с лёгкой тенью.

## Файлы

| Файл | Что | Размеры |
|------|-----|---------|
| `app_icon.png` | PNG высокого разрешения (источник/превью) | 512×512 |
| `app_icon.ico`, `app_icon_high.ico` | Иконки приложения **M4 CE** | 16, 32, 48, 64, 128, 256 |
| `brand/m4-crm/app_icon.png`, `.ico` | Иконки для приложения **M4 CRM** | те же |

`app_icon_high.ico` — та, что реально используется в сборке и коде.

## Дизайн

- Монограмма: строка 1 «M4» (бордо), строка 2 — код приложения («CE» / «CRM», белым).
- **Мультиразмерный .ico со сменой отрисовки под размер:** на 256/128/64/48 — полная
  монограмма; на 32 — «M4» крупно + плашка-подпись; на 16 — только «M4» во всю плитку
  (подпись в 16 px всё равно нечитаема, поэтому убрана ради чёткости). Так иконка
  остаётся фирменной крупно и читаемой в трее и заголовке окна.

### Палитра

| Роль | HEX |
|------|-----|
| Градиент, верх | `#FF9D3D` |
| Градиент, низ | `#E4670E` |
| «M4» (строка 1) | `#8E1C10` |
| Код приложения (строка 2) | `#FFFFFF` |

## Как перегенерировать

```
python create_icon.py
```

Правки стиля/палитры/списка приложений — в самом `create_icon.py` (константы вверху и
`main()`). Скругление плитки — `RADIUS_FRAC`, качество краёв — `SS` (суперсэмплинг).

## Интеграция (имена файлов не меняются при регенерации)

- Сборка: `src/pysidedeploy.spec` → `icon = …/app_icon_high.ico`
- Точка входа: `src/app.py` → `QApplication.setWindowIcon(app_icon_high.ico)`
- Главное окно: `src/light/main_window.py` → `setWindowIcon(app_icon_high.ico)`
